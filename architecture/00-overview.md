
# SOC Lab Architecture Overview

This document explains the architecture of the **Enterprise AD SOC Lab**.

The lab simulates a corporate network monitored by a Security Operations Center (SOC) using **Wazuh SIEM** and **Security Onion IDS**.

The environment includes Windows and Linux systems, endpoint telemetry, network monitoring, and an attacker machine.

---

# Network Overview

Primary SOC Network

192.168.117.0/24

All monitored systems operate within this network.

# System Roles

## Wazuh Manager

Central SIEM platform.

Functions:

- collect endpoint logs
- correlate security events
- detect suspicious activity
- generate SOC alerts

Installed on:

Ubuntu server

IP:

192.168.117.50

---

## Domain Controller (DC01)

Windows Server 2025.

Functions:

- Active Directory
- DNS
- DHCP
- authentication services

Security telemetry includes:

- Windows Security Logs
- Sysmon telemetry
- PowerShell logs

---

## Windows 11 Client

Represents a typical enterprise endpoint.

Functions:

- domain workstation
- user activity simulation
- attack target

Telemetry collected:

- Sysmon logs
- Windows event logs
- PowerShell activity

---

## Ubuntu Server

Linux server monitored by Wazuh.

Telemetry collected:

- SSH authentication logs
- Linux system logs
- file monitoring

Log source:

/var/log/auth.log

---

## Security Onion

Network Security Monitoring platform.

Components:

- Suricata (IDS)
- Zeek (network metadata)
- packet capture

Functions:

- detect network scanning
- detect suspicious connections
- detect attack signatures

---

## Kali Linux

Attacker simulation machine.

Used for:

- network reconnaissance
- brute force attacks
- exploitation simulations
- lateral movement simulations

Tools used:

- nmap
- hydra
- smbclient
- netcat
- custom scripts

# Monitoring Architecture

The SOC monitoring pipeline works as follows.

Endpoint Activity  
↓  
Sysmon logs events  
↓  
Wazuh agents collect logs  
↓  
Logs sent to Wazuh SIEM  
↓  
Security Onion monitors network traffic  
↓  
SIEM correlates alerts  
↓  
SOC analyst investigates events

---

# Telemetry Sources

## Endpoint Telemetry

Collected using:

Sysmon

Events include:

- process creation
- network connections
- registry changes
- file creation

---

## Windows Logs

Collected via Wazuh agent.

Important events include:

4624 — Successful login  
4625 — Failed login  
4728 — Privileged group membership change

---

## Linux Logs

Collected from:

/var/log/auth.log

Events include:

SSH authentication attempts

---

## Network Telemetry

Collected by Security Onion.

IDS alerts generated by:

Suricata

Network metadata recorded by:

Zeek

Example detection pipeline.

Attacker executes command from Kali
