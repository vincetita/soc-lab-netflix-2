# Attack Simulation 2 — Brute Force Attacks

This simulation demonstrates credential brute force attacks within the SOC lab.

Two scenarios are simulated:

1. SSH brute force attack against the Ubuntu server
2. Windows failed logon attempts against Active Directory

Both activities generate authentication failure logs which are detected by the SOC monitoring stack.

MITRE ATT&CK Technique

T1110  
Brute Force

---

# Lab Systems

| System | Role | IP |

---

# Scenario 1 — SSH Brute Force (Linux)

## Attack Objective

Attempt to guess the SSH password of the Ubuntu server using a password dictionary.

---

# Attack Execution

From Kali Linux run:


hydra -l root -P /usr/share/wordlists/rockyou.txt ssh://192.168.117.129


Explanation

-l root  
Target username

-P rockyou.txt  
Password wordlist

ssh://192.168.117.129  
Target SSH service

Hydra rapidly attempts multiple login combinations.

---

# Logs Generated on Ubuntu

Ubuntu records authentication attempts in


/var/log/auth.log


Example log entry


Failed password for root from 192.168.117.X port 42321 ssh2


These repeated failures indicate a brute force attack.

---

# Detection — Wazuh

The Wazuh agent installed on the Ubuntu server monitors authentication logs.

Wazuh detects patterns such as

• repeated login failures  
• authentication attempts from same IP  

Example alert


Multiple SSH authentication failures detected


Severity

Medium

---

# Detection — Security Onion

Network activity generated by Hydra is detected by

Suricata IDS  
Zeek network logs

Indicators include

• repeated SSH connection attempts  
• unusual authentication traffic

---

# SOC Investigation

SOC analyst workflow

1 Review Wazuh alert

2 Identify attacker IP

3 Inspect Ubuntu auth.log

4 Confirm repeated login failures

5 Determine if brute force attack is ongoing

# Scenario 2 — Windows Failed Logon (Active Directory)

## Attack Objective

Simulate failed authentication attempts against the domain controller.

These generate Windows Security Event ID 4625.

---

# Attack Execution

From Win11Client run:


runas /user:corp\administrator cmd


Windows will prompt for a password.

Enter an incorrect password.

Repeat the command multiple times.

This simulates a credential brute force attack.

---

# Windows Logs Generated

The domain controller records the event.

Log source

Windows Security Log

Event ID

4625

Event message


An account failed to log on


Important fields

• Account name  
• Source network address  
• Logon type  

Example source IP


192.168.117.100


---

# Detection — Wazuh

The Wazuh agent installed on DC01 collects Windows event logs.

Example rule


Windows failed logon detected
Rule ID: 18107


Alert severity

Medium

---

# Detection — Sysmon

Sysmon provides additional telemetry including

• process execution  
• authentication context  
• network connections

Log location


Microsoft-Windows-Sysmon/Operational


---

# SOC Investigation

SOC analyst workflow

1 Identify Wazuh alert

2 Review Windows Event ID 4625

3 Identify source IP

4 Confirm repeated login failures

5 Determine attack origin

Result

The source host is identified as **Win11Client**.
